Présentation rapide

Ce projet est conçu pour être construit automatiquement, testé et déployé à chaque push sur la branche main (ou celle que vous aurez configurée).

  • Tests unitaires et linting dans l'étape CI.
  • Build artefact + image Docker (si nécessaire).
  • Push vers registre et déploiement automatique (CD).

Workflow recommandé

Commit & PR
Branch feature → PR vers main
CI - Tests & Lint
Exécuter tests unitaires, intégration rapide et analyse statique
Build
Packaging, build Docker image, génération d'artefacts
CD - Déploiement
Déployer sur staging puis production après validations

Extrait de configuration (GitHub Actions)

# .github/workflows/ci-cd.yml
name: CI/CD
on:
  push:
    branches: ["main"]
  pull_request:
    branches: ["main"]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Setup
        run: echo "Setup, install deps..."
      - name: Tests
        run: echo "pytest -q"
      - name: Build
        run: echo "docker build -t ghcr.io/${{ github.repository }}:latest ."
      - name: Publish
        if: github.ref == 'refs/heads/main'
        run: echo "push image to registry / trigger deploy"

Quick start

  1. Forker le dépôt et créer une branche feature.
  2. Ouvrir une PR; CI s'exécutera automatiquement.
  3. Après merge vers main, CD déploiera sur staging, puis prod si configuré.
